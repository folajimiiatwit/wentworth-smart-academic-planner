# Lessons Learned

- Real-world documents require defensive parsing rather than one perfect regular expression.
- Visual PDF appearance and extracted text structure can differ significantly.
- Small test transcripts should be added whenever a new parsing bug is discovered.
- Frontend and backend responsibilities should remain separated.
- AI output must be validated by deterministic degree and prerequisite rules.
- Clear configuration and error messages improve maintainability.
- Documentation should be updated at the same time as the implementation.
